---
title: "OHLCV WebSocket"
description: "Live partial OHLCV bars plus closed-bar finalizations over WebSocket. Drives chart UIs without polling."
---

`/ws/ohlcv` pushes both the in-progress (partial) bar and the final (closed) bar for the timeframes you subscribe to. Use it to drive a candlestick chart without polling [`GET /api/v1/candles/latest`](/api-reference/candles-ohlcv/listlatestcandles).

## Endpoint and auth

```
wss://ws.dexploit.dev/ws/ohlcv
```

Authentication is the same as [`/ws/swaps`](/streaming/websocket#endpoint-and-auth) — `Authorization: Bearer ohlcv_live_sk_<key>` header, or `?api_key=…` query string.

## Subscribe

Send a subscribe message immediately after the welcome frame:

```json
{
  "pair_addresses": ["<pool_address>"],
  "timeframes": ["1m", "5m"]
}
```

`pair_addresses` is required and must be non-empty; you can subscribe to many pools at once. `timeframes` accepts the same set as the REST endpoints: `1s`, `30s`, `1m`, `5m`, `15m`, `1h`, `4h`, `1d`. The server replies with an ack:

```json
{ "status": "subscribed", "pairs": 1, "timeframes": ["1m", "5m"], "subjects": 2 }
```

**Legacy `mints` / `protocols` payloads are rejected** — the error message points you at `/api/v1/pairs?token_address=X` to discover the right `pair_address`.

## Frame shape

Each bar update arrives as a `type: "ohlcv"` frame:

```json
{
  "type": "ohlcv",
  "pair_address": "...",
  "token_address": "...",
  "timeframe": "1m",
  "timestamp": "2026-05-19T22:14:00+00:00",
  "open":  0.000635,
  "high":  0.000641,
  "low":   0.000632,
  "close": 0.000638,
  "volume_sol":    12300000000,
  "volume_token":  19340000,
  "trade_count":   42,
  "buy_count":     27,
  "sell_count":    15,
  "unique_traders": 31,
  "is_closed": false
}
```

A few things worth flagging:

- **`is_closed` is the signal you want.** `false` means this is an update to the currently-forming bar; the same `(pair_address, timeframe, timestamp)` will receive multiple `is_closed: false` frames as trades arrive. The bar is final the first time you see it with `is_closed: true`.
- **`volume_sol` is in lamports** (1 SOL = 1e9). Divide by 1e9 for human-readable SOL.
- **`volume_token` is in base units.** Divide by `10 ** token_decimals` (look up via [`GET /tokens/{mint}`](/api-reference/tokens/gettoken)).
- **`timestamp`** is the bar's open time as RFC3339 — the bar covers `[timestamp, timestamp + timeframe)`.

## Replay-then-stream pattern

For chart UIs, the standard pattern is:

1. Call [`GET /api/v1/candles/latest`](/api-reference/candles-ohlcv/listlatestcandles) for an initial backfill.
2. Open `/ws/ohlcv` and subscribe to the same `(pair_address, timeframe)`.
3. On each frame: if `is_closed: true`, append (replacing any prior partial at the same timestamp). If `is_closed: false`, replace the current head bar in place.

Gap-fill via REST if you disconnect — the same approach as [Reconnect & backpressure](/streaming/reconnect-backpressure) for `/ws/swaps`.
