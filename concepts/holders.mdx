---
title: "Holders"
description: "Two different holder counts exist — snapshot-driven and swap-derived. Here's when each updates and which one to trust."
---

Dexploit exposes two holder-related numbers that look the same but are computed differently. Knowing which is which avoids the most common "why don't these match?" question.

## Snapshot-driven (`/tokens/{mint}/holders` + `/tokens/{mint}/concentration`)

A periodic job walks the SPL token-account state for the mint and writes a snapshot to ClickHouse. [`GET /tokens/{mint}/holders`](/api-reference) and [`GET /tokens/{mint}/concentration`](/api-reference) both read from the most recent snapshot.

- **Freshness:** snapshots are refreshed automatically on a schedule, and on-demand via [`POST /tokens/{mint}/snapshot-now`](/api-reference). `snapshot_ts` (Unix seconds) tells you exactly how old the data is.
- **Cold mints:** when nothing has snapshotted a mint yet, both endpoints return placeholder shapes with `total_holders: 0` (or an empty `holders` array). That's not "no holders" — it's "no snapshot yet". POST `/snapshot-now` and poll back in ~35 seconds.
- **Truncation:** for mints with very large holder sets the snapshot may be capped — `is_partial: true` on `/holders` indicates this.

`top10_pct`, `top100_pct`, `gini`, and `total_holders` on `/concentration` are all derived from the snapshot.

## Swap-derived (`/tokens/{mint}/intel.holders`)

[`GET /tokens/{mint}/intel`](/api-reference) returns a `holders` field too — but it's computed differently. The intel handler counts distinct wallets that have *traded* the mint on a DEX in the recent past. It's cheaper to compute (no on-chain walk) but answers a slightly different question.

**Expect these two numbers to diverge.** A mint can have 5,000 SPL holders but only 1,200 distinct on-DEX traders if most holders airdropped in. Use the snapshot number for "how many people own this", and the intel number for "how active is this".

## When to use which

| You want to know… | Use |
| --- | --- |
| Total wallets that hold the SPL token | `/tokens/{mint}/concentration` → `total_holders` |
| % of supply concentrated in top 10 | `/tokens/{mint}/concentration` → `top10_pct` |
| Full holder list (paginated) | `/tokens/{mint}/holders` |
| Distinct on-DEX traders, lower-latency | `/tokens/{mint}/intel` → `holders` |
| Force a fresh snapshot before reading | `POST /tokens/{mint}/snapshot-now`, then poll |
